# 在下载节点后添加5秒休眠

## 需求
在每次点击下载之后，休眠5秒，避免同时下载多个图片时出现问题。

## 解决方案
在"点击下载对应的图片"节点之后添加一个**Wait节点**，设置等待时间为5秒。

## 操作步骤

### 1. 添加Wait节点
1. 在n8n工作流编辑器中，找到"点击下载对应的图片"节点
2. 点击节点右侧的"+"按钮，添加新节点
3. 搜索并选择"Wait"节点

### 2. 配置Wait节点
在Wait节点的配置中：
- **Wait Type**: 选择 "Wait for Time"（等待固定时间）
- **Amount**: 设置为 `5`
- **Unit**: 选择 "Seconds"（秒）

### 3. 连接节点
- 将"点击下载对应的图片"节点的输出连接到Wait节点
- Wait节点的输出可以连接到后续节点（如果有的话）

## 节点配置示例

Wait节点的JSON配置应该是这样的：

```json
{
  "parameters": {
    "amount": 5,
    "unit": "seconds"
  },
  "type": "n8n-nodes-base.wait",
  "typeVersion": 1.1,
  "position": [608, 464],
  "name": "等待5秒"
}
```

## 工作流流程

修复后的流程：
1. 检查是否存在需要下载结果的Tab
2. 提取需要下载的图片
3. Split Out（拆分多个图片）
4. If（判断是否匹配）
5. **点击下载对应的图片** ← 这里
6. **等待5秒** ← 新增
7. （继续处理下一个图片，如果有的话）

## 注意事项

- 由于使用了Split Out节点，每个图片都会单独处理
- 每次点击下载后都会等待5秒，然后再处理下一个
- 这样可以避免同时触发多个下载操作，导致浏览器或下载服务过载
